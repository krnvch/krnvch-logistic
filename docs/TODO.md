### 10.8 🟡 Аудит безопасности платформы + GitHub-репозитория

Провести полный аудит безопасности приложения и публичного GitHub-репозитория. Вызвать агента Security Engineer (#14) для проверки:

**Код и данные:**

- Поиск утечек секретов (API-ключи, пароли, токены) в коде и git-истории
- Проверка `VITE_`-переменных — что в клиентский бандл попадают только публичные значения
- Проверка `.gitignore` — все чувствительные файлы (`.env`, ключи) исключены
- Аудит зависимостей (`pnpm audit`) — нет ли пакетов с известными уязвимостями

**Авторизация и доступ:**

- Ревью RLS-политик Supabase — правильно ли разделены роли operator/worker
- Проверка на privilege escalation — может ли worker обойти ограничения через прямые запросы
- Валидация auth-флоу — хранение токенов, сессии, refresh, logout

**Уязвимости (OWASP Top 10):**

- XSS — нет ли `dangerouslySetInnerHTML`, небезопасной интерполяции
- Injection — все запросы через Supabase SDK (параметризованные)
- Утечка информации — ошибки, console.log, stack traces не раскрывают внутренности

**GitHub-репозиторий (публичный!):**

- Сканирование всей git-истории на случайно закоммиченные секреты
- Проверка GitHub Actions — секреты, pinning actions, permissions
- Проверка, что preview-деплои Vercel не раскрывают чувствительные данные

**Чему научишься:** application security (AppSec), threat modeling, OWASP Top 10, секрет-менеджмент, аудит зависимостей, как защищать публичные репозитории, RLS как уровень безопасности.

### 10.7 🟡 Переключение вида: таблица ↔ Kanban-доска

Дать пользователю возможность переключать отображение рейсов (активных/архивных) между табличным видом и Kanban-доской (колонки по статусам). Сохранять выбранный вид в localStorage.

**Чему научишься:** паттерн переключения view mode, Kanban UI, компонентная архитектура (два представления одних данных), localStorage-персистенция, drag & drop (опционально).

### 10.9 🟡 Лендинг / маркетинговый сайт

Создать отдельный одностраничный сайт (landing page), который продвигает платформу: объясняет преимущества, описывает USP (unique selling point), показывает скриншоты/демо. Отдельный URL (не внутри основного приложения). CTA-кнопки (регистрация, демо, контакт). По сути — классический маркетинговый сайт для веб-приложения.

**Чему научишься:** маркетинговый сайт vs продуктовый сайт, landing page best practices, copywriting для продукта, CTA-стратегия, SEO basics, деплой отдельного проекта, как продукты «продают себя» через сайт.
