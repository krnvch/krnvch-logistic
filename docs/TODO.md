## 1. Тестирование

> *"Как разработчики убеждаются, что ничего не сломалось?"*

### ~~1.1 🟢 Unit-тесты (Vitest)~~ ✅

~~Написать первые тесты для функции `getOrderStatus()` — она чистая, без зависимостей, идеальна для старта.~~

**Чему научишься:** что такое unit-тест, test runner, assertions, TDD-подход (сначала тест → потом код).

> **Выполнено:** 10 тестов для `getOrderStatus()` (Vitest). PR [#2](https://github.com/krnvch/tulip-logistic/pull/2). См. `docs/learning-log.md`.

### 1.2 🟡 Component-тесты (Testing Library)

Проверить, что `OrderCard` показывает правильные кнопки для оператора и работника.

**Чему научишься:** как тестируют UI-компоненты, что такое render/screen/userEvent, моки (mocks).

### 1.3 🔴 E2E-тесты (Playwright)

Полный сценарий: логин → создание рейса → добавление заказа → размещение на стене → отметка "Готово".

**Чему научишься:** end-to-end тестирование, Page Object Model, CI-пайплайн с тестами, headless-браузеры.

## 3. Работа с базой данных

> *"Как менять структуру БД, не ломая прод?"*

### 3.1 🟡 Миграция: добавить поле `priority` к заказам

Заказы бывают срочные. Добавить поле `priority` (normal/urgent) — через миграцию Supabase.

**Чему научишься:** database migrations, backward compatibility, как менять схему на живом проде.

### 3.2 🟡 Миграция: история действий (audit log)

Таблица `activity_log` — кто, когда, что сделал (создал заказ, переместил коробки, отметил готово).

**Чему научишься:** аудит-логи, триггеры БД, зачем нужна трассировка действий.

## 4. UX/UI фичи

> *"Фичи, знакомые дизайнеру — но теперь ты увидишь их изнутри"*

### 4.1 🟢 Тёмная тема

Добавить переключатель светлая/тёмная тема. shadcn/ui уже поддерживает — нужно подключить.

**Чему научишься:** CSS-переменные, `prefers-color-scheme`, localStorage, theming system.

### 4.2 🟢 Индикатор онлайн-пользователей

Показать, кто сейчас в системе (зелёная точка рядом с именем).

**Чему научишься:** Supabase Presence (Realtime), WebSocket, как работает "онлайн-статус".

### 4.3 🟡 Drag & Drop для размещений

Перетаскивать заказы из сайдбара на стены трейлера мышкой.

**Чему научишься:** drag-and-drop API, библиотека dnd-kit, UX перетаскивания, touch-события.

### 4.4 🟡 Push-уведомления

Работник получает уведомление, когда оператор добавляет новый заказ.

**Чему научишься:** Web Push API, Service Workers, разрешения браузера, notification UX.

## 5. Продуктовые практики

> *"Как продакты и дизайнеры принимают решения на основе данных?"*

### 5.1 🟡 Аналитика действий

Трекать ключевые события: логин, создание заказа, завершение рейса. Подключить Posthog или Mixpanel.

**Чему научишься:** event tracking, product analytics, воронки, retention, data-driven decisions.

### 5.2 🟡 Feature Flags

Добавить возможность включать/выключать фичи без деплоя (через env-переменные или сервис типа LaunchDarkly).

**Чему научишься:** feature flags, canary releases, A/B тестирование, gradual rollout.

---

## 6. Производительность и качество

> *"Как делают приложения быстрыми и надёжными?"*

### 6.1 🟢 Lighthouse-аудит и оптимизация

Прогнать Lighthouse, исправить замечания по Performance, Accessibility, SEO.

**Чему научишься:** Core Web Vitals, Lighthouse scoring, a11y (accessibility), как Google оценивает сайты.

### 6.2 🟡 PWA (Progressive Web App)

Сделать приложение устанавливаемым на телефон — иконка на рабочем столе, работа без интернета.

**Чему научишься:** Service Worker, manifest.json, offline-first, кэширование, app shell pattern.

### 6.3 🟡 Error Monitoring (Sentry)

Подключить Sentry — чтобы видеть ошибки пользователей в реальном времени.

**Чему научишься:** error tracking, stack traces, source maps, alerting, как девы узнают о багах.

## 8. Масштабирование

> *"Что делают, когда проект растёт?"*

### 8.1 🟡 Мультиязычность (i18n)

Добавить английский язык + переключатель. Все строки — из файлов переводов.

**Чему научишься:** интернационализация, i18n-библиотеки (react-intl / i18next), locale, RTL.

### 8.2 🔴 Мульти-тенантность (несколько компаний)

Каждая логистическая компания — отдельный "tenant" со своими данными.

**Чему научишься:** multi-tenancy, RLS (Row Level Security), organization-based access, SaaS-архитектура.
